Rx Pipeline — 수신 경로

대기 상태에서도 오디오 디코더를 즉시 재생 가능한 상태로 유지하여, 발화자 전환 시 끊김 없는 오디오 연속성을 보장합니다.

Audio Decoder Warm-keeping

미디어 엔진의 오디오 디코더는 입력이 없으면 절전 모드로 전환됩니다. 이 상태에서 실제 음성이 도착하면 디코더 웜업에 수십 ms가 소요되어 첫 음절이 잘리는 현상이 발생합니다.

문제 디코더 절전 → 복귀 지연 (모드 전환 비용)
해결 대기 중 최소 비용의 무음 프레임을 로컬 주입하여 디코더를 "깨어있는" 상태로 유지
기대 첫 음성 지연 최소화 (목표 ~0ms), 배터리 영향 무시 가능

Seamless Stream Continuity

발화자가 전환되면 새로운 미디어 스트림이 시작됩니다. 이때 디코더가 유지하던 시퀀스와 새 스트림의 시퀀스 사이에 불연속이 발생하면 디코더가 이를 대규모 패킷 손실로 오인합니다.

문제 스트림 전환 시 시퀀스 불연속 → 디코더 오동작
해결 미디어 엔진의 공식 확장 인터페이스를 활용한 무손실 패킷 보정 레이어 — SFU 서버의 검증된 로직을 클라이언트로 이식
결과 발화자 전환 시 끊김 없는 연속 재생
Video Rx — 비디오는 패킷 레벨 조작을 배제합니다. 키프레임 확보는 아래 Client-driven Readiness 프로토콜에서 처리합니다.

연결 유지 신호 주기를 극단적으로 늘려 모뎀 웨이크업을 최소화합니다. 미디어 엔진 자체는 내리지 않습니다 — 암호화 키와 인증서가 메모리에서 소실되면 전체 보안 핸드셰이크를 재수행해야 하기 때문입니다.

NAT 매핑 소실 가능 — 장주기 유지 시 네트워크 경로가 만료될 수 있습니다. 이 경우 연결 재협상이 필요하며, 10분+ 미사용자에게 1~2초 복구는 허용 가능한 수준입니다.

NAT Traversal Optimization

미디어 미전송 상태에서도 연결 유지 신호가 주기적으로 모뎀을 깨웁니다. FSM 단계별로 신호 주기를 동적 조정하여 배터리 소모를 최소화합니다.

실측 필수 — 국내 주요 통신사(SKT / KT / LGU+)의 LTE / 5G 환경별 NAT 타임아웃은 20~60초로 편차가 큽니다. 연결 유지 주기는 실측 데이터 기반으로 튜닝합니다.

Client-driven Readiness Protocol

카메라 콜드 스타트를 극복하기 위해, 서버가 임의 타이밍에 키프레임을 요청하는 대신 클라이언트가 준비 완료를 선언하는 주도적 프로토콜을 사용합니다.

기존 방식의 문제

서버가 타이밍을 추측하여 키프레임 요청을 보냅니다. 카메라 AE/AWB 수렴 전에 요청이 도착하면 허공에 낭비되고, 수렴 후 한참 뒤에 도착하면 불필요한 지연이 발생합니다.

Client-driven 방식의 이점

클라이언트가 "첫 프레임 캡처 완료" 콜백을 받은 시점에 서버에 준비 완료를 알립니다. 서버의 타이밍 추측이 불필요하며, 첫 프레임 캡처 = AE 1차 수렴이므로 키프레임 요청 2발이면 충분합니다.

Exception Handling — Signaling Disconnection

시그널링 채널(TCP)이 단절되면, 모든 캐시된 상태를 즉시 파기하고 완전한 재시작을 수행합니다.

Zero Tolerance — Full Cold Start

시그널링 연결이 유실되면 SDP, 암호화 컨텍스트, 연결 상태 등 모든 캐시를 즉시 파기합니다.

이후 이벤트 발생 시 완전히 새로운 세션을 수립합니다 — 서버-클라이언트 상태 불일치(좀비 세션)를 원천 차단합니다.

타협 없음.
